# 账号系统与 Token 隔离设计

本系统采用账号名、密码登录。登录成功后服务端生成临时 token，客户端后续请求通过 `X-Auth-Token` Header 或 `auth-token` 查询参数携带 token。

## 1. 账号目录

账号目录由账号名推导：

```text
storage/accounts/
  └── {账号名前三位}_{SHA256(账号名)前六位}/
      ├── account.json
      ├── account_tokens.jsonl
      ├── profile.json
      ├── analysis_params.json
      └── runs/
          ├── diagnosis/
          │   └── {run_id}/
          └── price_recommendation/
              └── {run_id}/
```

`account.json` 只保存账号元信息和 `passwordHash`，密码使用 bcrypt 慢哈希保存，不保存明文密码，不保存旧凭证。

## 2. 登录与 Token

登录接口：

| 接口 | 方法 | 说明 |
| :--- | :--- | :--- |
| `/api/auth/register` | `POST` | 创建账号，写入 bcrypt 密码哈希，并返回登录 token |
| `/api/auth/login` | `POST` | 使用账号名和密码登录，返回登录 token |
| `/api/auth/verify` | `POST` | 校验当前 token 是否有效 |
| `/api/auth/service-token` | `POST` | 使用用户 token 创建客服 token |
| `/api/auth/logout` | `POST` | 撤销当前 token |

普通 token 格式：

```text
{账号名前三位}_{SHA256(账号名)前六位}_{随机串}
```

客服 token 格式：

```text
serv_{账号名前三位}_{SHA256(账号名)前六位}_{随机串}
```

服务端只保存 token 的 SHA256 哈希，不保存 token 明文。客户端必须保存并回传完整 token，后端从完整 token 中解析账号目录前缀，再读取该账号目录下的 `account_tokens.jsonl` 校验状态。

对外接口和 Web App 只展示账号名前三位加星号，例如 `abc***`。该展示字段不是认证凭证，也不参与账号定位；目录名和 token 前缀仍使用账号名前三位加账号名哈希前六位，用于服务端定位账号目录。

## 3. Token 生命周期

普通用户 token：

- 软过期：距离上次使用超过 2 小时则失效。
- 硬过期：距离创建超过 12 小时则强制失效。

客服 token：

- 软过期：距离上次使用超过 30 分钟则失效。
- 硬过期：距离创建超过 2 小时则强制失效。

token 校验通过后，服务端会追加一条 `active` 事件，刷新软过期时间。发现硬过期时，会清理该 token 相关历史事件，避免日志无限膨胀。

## 4. Token 事件日志

每个账号目录下的 `account_tokens.jsonl` 记录 token 状态变更：

```json
{"action":"creation","time":"2026-06-09T10:00:00+00:00","token_hash":"..."}
{"action":"active","time":"2026-06-09T10:10:00+00:00","token_hash":"..."}
{"action":"serv_creation","time":"2026-06-09T10:20:00+00:00","token_hash":"...","parent_token_hash":"..."}
{"action":"revoke","time":"2026-06-09T10:30:00+00:00","token_hash":"..."}
```

校验时倒序读取日志。若最近一条匹配记录是 `revoke`，该 token 立即无效。

## 5. 客服 Token 权限

客服 token 在接口内部按操作类型限制：

- 允许：查看状态、读取历史、上传文件、启动任务、读取报告、下载产物。
- 禁止：删除类操作、修改安全设置类操作。

客服权限不使用全局开关控制。

## 6. 前端登录态

Web App 登录成功后把 token 存入 `sessionStorage.authToken`。浏览器会话结束后 token 不再保留。

如果使用 token 调接口时后端返回 `401`，前端清除本地登录态并跳回 `/login.html`。

## 7. 管理员链路

管理员后台继续使用独立的 `X-Admin-Token`。该 Header 只服务 `/api/admin/*` 管理接口，不参与用户账号登录、用户 token 校验或客服 token 权限判断。
