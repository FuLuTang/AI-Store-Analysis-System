# Smolagents CodeAgent — PoC 对照实验

> 目标：快速验证 CodeAgent 对脏数据 / 嵌套 JSON / 不规则输入的处理能力

---

## 定位

**不追求生产质量**。只做一条端到端通路验证：

```
输入样例 → CodeAgent 写 Python 展平 → DuckDB 查询 → 输出 manifest/metrics
```

如果 PoC 结果好（处理脏数据明显强于 Pydantic AI），可以将它的 Python 代码生成能力拆成解析工具子模块，接到 Pydantic AI 管线上。

如果只是 demo 里骚，生产接入麻烦 → 弃用。

## PoC 范围

最小闭环，不做：

- ✗ 多步编排 / AgentService
- ✗ 结构化输出校验
- ✗ FastAPI 路由
- ✗ 错误重试 / SSE 日志

只做：

- ✓ 一个 Python 入口函数，喂入脏数据
- ✓ CodeAgent 自己写 Python 展平
- ✓ 写入 DuckDB 并跑 SQL
- ✓ 输出指标 JSON

## 测试数据

优先用仓库现有的脏数据：

```
data/examples/
  └── 概览-日.json      (多层嵌套)
  └── 概览-月.json      (多层嵌套)
  └── o2o营业-日.json    (嵌套 + 字段别名)
  └── 商品资料.csv       (纯表格，字段多)
```

额外自造一个 `{{{[[[]]]]}}}` 深度嵌套 JSON 来测极限。

## 评判标准

| 维度 | 期望 |
|------|------|
| 嵌套 JSON 展平 | 自己写递归展平，不依赖外部库 |
| 脏数据鲁棒性 | 空值 / 类型混用 / 错位字段能兜底 |
| SQL 生成 | 能用 DuckDB SQL 完成 ratio / group by 等基本指标 |
| 代码可靠性 | 同一输入多次跑，生成的代码和结果是否稳定 |

## 如果成功（决定合并到主框架）

把 CodeAgent 的 Python 代码生成能力抽取为：

```
packages/core/agent_tools/
  └── codegen_parser.py   ← 输入原始数据 → 输出 DataFrame
  └── codegen_mapper.py   ← 输入字段列表 → 输出字段映射
```

主框架仍走 Pydantic AI，但解析/映射两步可换成 CodeAgent 生成代码执行。
