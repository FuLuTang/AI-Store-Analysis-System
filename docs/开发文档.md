# 开发文档 (Developer Documentation)

## 1. 架构概览
本项目采用 **Monorepo** 结构，通过 `npm workspaces` 管理多个子项目和包。

### 核心流程
1. **数据输入**: 用户通过 `apps/web` 上传 JSON 数据或加载示例。
2. **清洗与计算**: `apps/api` 调用 `packages/core/cleaner.js` 清洗数据，并使用 `packages/core/metrics.js` 计算经营指标。
3. **AI 诊断**: 
   - 第一阶段：调用 `packages/ai/ai-caller.js` 生成基础经营报告。
   - 第二阶段：调用 `packages/ai/error-reviewer.js` 进行异常复核与深度分析。
4. **结果输出**: 合并生成 Markdown 报告，存储在 `storage/reports/` 并流式反馈给前端。

## 2. 核心模块开发

### 添加新指标 (Metrics)
在 `packages/core/metrics.js` 中添加新的计算函数。每个函数应返回以下格式：
```javascript
{
  id: "metric_id",
  name: "指标名称",
  value: 123.45,
  status: "warning", // "normal", "warning", "critical"
  detail: "详细描述..."
}
```

### 修改 AI 提示词 (Prompts)
提示词模板维护在 `packages/ai/` 目录下的对应模块中。修改时请注意保持 JSON Schema 的一致性。

## 3. 目录规范
- **`apps/api/src/`**: 业务逻辑控制器与路由。
- **`packages/core/`**: 纯逻辑函数，不应包含 I/O 操作（由 API 层传入数据）。
- **`storage/`**: 该目录已被加入 `.gitignore`，仅用于存放运行时产生的临时文件。

## 4. 环境配置
复制 `.env.example` 到 `.env` 并配置您的 API Key：
```bash
OPENAI_API_KEY=your_key_here
OPENAI_BASE_URL=https://api.openai.com/v1
```

## 5. 调试建议
- **API 调试**: 观察 `apps/api/src/server.js` 的终端输出，系统会实时打印各节点的执行日志。
- **数据追踪**: 生成的中间数据会缓存在 `storage/cache/` 下，方便复溯分析。
