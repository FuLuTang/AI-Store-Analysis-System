# 开发文档 (Developer Documentation)

## 1. 架构概览 (Python Core)
本项目已全面迁移至 **Python/FastAPI** 架构。采用计算与展示分离的设计，核心算法由 Pandas 驱动。

### 核心诊断流程
1. **数据输入**: 用户通过 `apps/web` 上传 JSON 数据。
2. **清洗预处理 (`packages/core/cleaner.py`)**: 使用 Pandas 进行数据标准化，推算等效天、等效月指标，处理 O2O 业务合并。
3. **指标计算 (`packages/core/metrics.py`)**: 
   - 核心营收与毛利趋势分析。
   - O2O 渠道占比与健康度计算。
   - 商品结构稳定性与热销品波动监测。
   - 生成结构化的 `anomaly_summary` (异常汇总)。
4. **AI 编排流 (`packages/ai/`)**: 
   - **初步诊断**: 根据清洗后的数据和异常指标生成第一版报告。
   - **审计复核**: 启动逻辑审计链，复核数据矛盾点，生成审计意见。
   - **报告融合**: 将算法日志、初诊报告、审计意见三方融合，生成最终的深度分析报告。
5. **实时反馈**: 通过 **SSE (Server-Sent Events)** 技术，将上述所有过程的日志实时流式传输至前端工作台。

## 2. 核心模块开发指南

### 添加新算法/指标 (Metrics)
在 `packages/core/metrics.py` 中添加新的计算函数。
- **推荐做法**: 使用 Pandas DataFrame 进行聚合计算。
- **返回值规范**: 函数应返回包含 `status` (normal/warning/critical) 和 `detail` 的字典，以便算法引擎自动汇总。

### AI 模型与 Prompt 维护
- **`ai_caller.py`**: 负责调用 OpenAI 兼容接口，支持流式和非流式切换。
- **Prompt 策略**: 我们采用了“任务拆解”模式，避免大模型在处理复杂长文本时产生幻觉。

## 3. 部署与环境

### 运行环境
- **Python 3.10+** (推荐 3.12/3.14)
- **FastAPI + Uvicorn** 作为 Web 容器
- **Pandas** 用于核心数学计算

### 目录权限
系统会自动管理 `storage/` 目录：
- `storage/uploads/`: 存放原始上传 JSON。
- `storage/cache/`: 存放处理过程中的临时数据。

## 4. 调试与监控
- **实时日志**: 运行 `npm run dev` 后，在浏览器控制台或前端“后台监控流”页面可以查看完整的 SSE 日志输出。
- **算法验证**: 可以在 `packages/core/` 下编写独立的测试脚本运行 metrics 函数，不依赖于 Web 接口。

## 5. 前端交互
前端位于 `apps/web/public`，通过 Fetch API 与后端 `/api/run` 交互，并通过 `EventSource` 监听 `/api/stream` 获取实时进度。
