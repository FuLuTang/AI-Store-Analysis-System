# 指标计算文档

> 本文档定义"算法处理"模块中所有指标的函数签名、输入、输出。
> 分为两部分：**A. 纯算法可算** / **B. 需AI辅助**

---

## 数据源类型定义

```ts
// 概览数据（日/月/年共用结构）
type OverviewData = {
  page: { viewType: 'day'|'month'|'year', selectedDate: string }
  summary: { metrics: Metric[] }
  ranking: { items: RankingItem[] }
  businessTable: { granularity: string, rows: BizRow[] }
  sourceDistribution: { center: {value:number}, items: SourceItem[] }
}

type Metric = {
  key: string, label: string, value: number, unit: string,
  rate?: number, rateUnit?: string,
  compare?: { value: number, unit: string, direction: string },
  extra?: { label: string, value: number, unit: string }
}

type BizRow = {
  period: string,
  retail_amount: number, gross_profit: number, visitor_count: number,
  member_amount: number, member_gross_profit: number,
  ecommerce_amount: number, ecommerce_gross_profit: number
}

// O2O数据
type O2OData = {
  page: { viewType: string }
  businessTable: { rows: O2ORow[] }
}

type O2ORow = {
  period: string,
  total_order_count: number, total_revenue: number,
  gross_profit: number, gross_margin: number,
  meituan_order_count: number, eleme_order_count: number,
  meituan_revenue: number, eleme_revenue: number,
  avg_order_value: number
}

// 热销数据
type HotProduct = {
  rank: number, product_name: string, barcode: string|null,
  sales_receipt_count: number|null, sales_quantity: number|null
}

type HotProductData = {
  page: { viewType: string }
  ranking: HotProduct[]
}

// 热销500数据
type Top500Product = {
  rank: number, product_name: string,
  sales_rank: number, city: string, barcode: string|null
}

type Top500Data = {
  page: { viewType: string }
  products: Top500Product[]
}
```

## 统一输出规范

所有指标函数的输出**必须**包含 `status` 字段：

```ts
type MetricStatus = 'pass' | 'attention' | 'warning' | 'uncountable'

// pass         — 正常，无需关注
// attention    — 值得注意，建议关注
// warning      — 异常/风险，需要行动
// uncountable  — 缺数据 / 无法计算 / 尚未实现
```

每个函数自行定义 pass/attention/warning 的阈值，但语义统一。
缺少必要输入数据、除以零、或函数未实现时，一律返回 `uncountable`。

## A. 纯算法可算指标

### A1. 经营基础类

---

#### `calcRevenueChange` ✅
**用途**：计算营收环比变化率

```
输入: rows: BizRow[]  (至少2行，按时间倒序)
输出: { current: number, previous: number, changePct: number, direction: 'up'|'down'|'flat', status: 'pass'|'attention'|'warning' }
```

算法：`changePct = (current - previous) / previous * 100`
status：|changePct| >50% → warning, >20% → attention, else pass

---

#### `calcGrossMargin` ✅
**用途**：计算毛利率

```
输入: grossProfit: number, revenue: number
输出: { marginPct: number, status: 'pass'|'attention'|'warning' }
```

算法：`marginPct = grossProfit / revenue * 100`
status：<15% → warning, <20% → attention, else pass

---

#### `calcAvgOrderValue` ✅
**用途**：计算客单价

```
输入: revenue: number, visitorCount: number
输出: { avgOrderValue: number, status: 'pass'|'attention'|'warning' }
```

算法：`avgOrderValue = revenue / visitorCount`
status：需结合历史均值判断，偏离>20% → attention, >40% → warning, else pass

---

#### `calcChannelMix` ✅
**用途**：计算渠道结构占比（普通/会员/电商）

```
输入: sourceDistribution: { items: SourceItem[] }
输出: {
  total: number,
  channels: { key: string, label: string, value: number, pct: number }[],
  dominant: string, dominantPct: number,
  status: 'pass'|'attention'|'warning'
}
```

算法：`pct = item.value / sum(all items) * 100`
status：单一渠道 >80% → warning, >60% → attention, else pass

---

#### `calcMemberPenetration` ✅
**用途**：会员渗透率（会员销售占比）

```
输入: memberRevenue: number, totalRevenue: number
输出: { penetrationPct: number, status: 'pass'|'attention'|'warning' }
```

算法：`penetrationPct = memberRevenue / totalRevenue * 100`
status：<5% → warning, <15% → attention, else pass

---

#### `calcMemberVsOverall` ✅
**用途**：会员客单价对比整体客单价

```
输入: memberAvgOrder: number, overallAvgOrder: number
输出: { memberAvg: number, overallAvg: number, diffPct: number, status: 'pass'|'attention'|'warning' }
```

算法：`diffPct = (memberAvg - overallAvg) / overallAvg * 100`
status：会员客单价低于整体 → attention, 低于整体50%以上 → warning, else pass

---

#### `calcDailyAvg`
**用途**：计算日均/月均营收和毛利

```
输入: total: number, days: number
输出: { dailyAvg: number, status: 'pass'|'attention'|'warning' }
```

算法：`dailyAvg = total / days`
status：日均低于历史均值30% → warning, 低于15% → attention, else pass

---

### A2. 趋势分析类

---

#### `calcConsecutiveChange` ✅
**用途**：计算连续增长或下降天数

```
输入: rows: BizRow[], field: 'retail_amount'|'gross_profit'|'visitor_count'
输出: { direction: 'up'|'down'|'flat', consecutiveDays: number, totalChangePct: number, status: 'pass'|'attention'|'warning' }
```

算法：从最近一行向前遍历，统计同方向连续变化次数
status：连续下降≥5天 → warning, ≥3天 → attention, else pass

---

#### `calcMovingAverage`
**用途**：N日移动平均

```
输入: rows: BizRow[], field: string, windowSize: number (默认7)
输出: { ma: number, values: number[], latestVsMa: number, status: 'pass'|'attention'|'warning' }
```

算法：`ma = sum(最近N个值) / N`
status：最新值低于MA 20% → warning, 10% → attention, else pass

---

#### `calcVolatility`
**用途**：波动率（变异系数）

```
输入: rows: BizRow[], field: string
输出: { mean: number, stdDev: number, cv: number, status: 'pass'|'attention'|'warning' }
```

算法：`cv = stdDev / mean`
status：cv >0.4 → warning, >0.25 → attention, else pass

---

#### `calcTrendSlope`
**用途**：趋势方向（线性回归斜率）

```
输入: rows: BizRow[], field: string
输出: { slope: number, direction: 'rising'|'falling'|'stable', status: 'pass'|'attention'|'warning' }
```

算法：简单线性回归 `y = ax + b`，返回 `a`
status：falling → attention, falling且斜率陡 → warning, else pass

---

#### `calcPeriodOverPeriod`
**用途**：任意两期环比

```
输入: currentValue: number, previousValue: number
输出: { changePct: number, direction: 'up'|'down'|'flat', status: 'pass'|'attention'|'warning' }
```

算法：`changePct = (current - previous) / previous * 100`
status：|changePct| >50% → warning, >20% → attention, else pass

---

#### `calcMaxDrawdown`
**用途**：最大回撤（从峰值到谷值的最大跌幅）

```
输入: rows: BizRow[], field: string
输出: { drawdownPct: number, peakPeriod: string, troughPeriod: string, status: 'pass'|'attention'|'warning' }
```

算法：遍历找峰值后的最大跌幅
status：回撤 >40% → warning, >20% → attention, else pass

---

#### `calcGrossMarginTrend` ✅
**用途**：毛利率变化趋势

```
输入: rows: BizRow[]
输出: { margins: {period: string, marginPct: number}[], avgMargin: number, latestMargin: number, trend: 'improving'|'declining'|'stable', slope: number, status: 'pass'|'attention'|'warning' }
```

算法：每期 `gross_profit / retail_amount * 100`，再算斜率方向
status：declining且最新<均值 → warning, declining → attention, else pass

### A4. 热销/动销类

---

#### `calcHotProductConcentration` ✅
**用途**：热销品集中度（TOP N 笔数占比）

```
输入: ranking: HotProduct[], topN: number (默认3)
输出: { topNReceiptCount: number, totalReceiptCount: number, concentrationPct: number, status: 'pass'|'attention'|'warning' }
```

算法：`concentrationPct = sum(前N名笔数) / sum(全部笔数) * 100`
status：TOP3占比 >70% → warning, >50% → attention, else pass

---

#### `calcProductStability` ✅
**用途**：爆品稳定性（跨时段排名变化）

```
输入: {
  today: HotProduct[], yesterday: HotProduct[],
  week: HotProduct[], month: HotProduct[]
}
输出: {
  stableProducts: string[],
  risingProducts: string[],
  fadingProducts: string[],
  summary: { stableCount, risingCount, fadingCount },
  status: 'pass'|'attention'|'warning'
}
```

算法：按 barcode 做4个时段的集合交叉对比
status：退热品 >3个 → warning, >1个 → attention, else pass

---

#### `calcBulkPurchaseItems`
**用途**：识别批量购买商品（数量/笔数 > 阈值）

```
输入: ranking: HotProduct[], threshold: number (默认1.5)
输出: { items: { name: string, ratio: number }[], status: 'pass'|'attention'|'warning' }
```

算法：`ratio = sales_quantity / sales_receipt_count`，ratio > threshold 则标记
status：有批量购买品 → attention（仅提示，非风险），else pass

---

#### `calcTop500Coverage`
**用途**：热销500覆盖率

```
输入: top500Total: Top500Product[], inStock: Top500Product[]
输出: { totalCount: number, inStockCount: number, coveragePct: number, status: 'pass'|'attention'|'warning' }
```

算法：`coveragePct = inStockCount / totalCount * 100`
status：覆盖率 <70% → warning, <85% → attention, else pass

---

#### `calcStockoutRate` ✅
**用途**：热销品头部缺货率（衡量门店核心商品的缺货情况）

```
输入: top500Total: Top500Product[], outOfStock: Top500Product[]
输出: { stockoutCount: number, headStockoutCount: number, stockoutPct: number, status: 'pass'|'attention'|'warning' }
```

算法：`stockoutPct = (outOfStock 中 sales_rank <= 50 的数量) / 50 * 100`
status：Top 50缺货率 >15% → warning, >5% → attention, else pass

---

#### `calcMissingCategoryRate` ✅
**用途**：热销品头部缺种率

```
输入: top500Total: Top500Product[], missingCategory: Top500Product[]
输出: { missingCount: number, headMissingCount: number, missingPct: number, status: 'pass'|'attention'|'warning' }
```

算法：`missingPct = (missingCategory 中 sales_rank <= 50 的数量) / 50 * 100`
status：Top 50缺种率 >10% → warning, >3% → attention, else pass

---

#### `calcHighRankStockoutAlert` ✅
**用途**：高排名缺货预警（城市热销排名靠前但缺货的品）

```
输入: outOfStock: Top500Product[], alertThreshold: number (默认50)

#### `calcHotMatchRate`

### A5. 异常检测类

#### `detectAnomalies`
**用途**：基于阈值/统计的异常检测

## B. 需AI辅助的指标

> 以下指标算法负责"算出数字"，但最终**判断/解读/标签**需要AI完成。
> 算法模块输出结构化数据，传给AI做最后一步。

---

#### `prepareStoreStatusLabel` ✅
**用途**：为AI准备门店状态标签的输入数据

```
输入: {
  revenueChange: number,       // 营收环比%
  grossMarginChange: number,   // 毛利率变化
  visitorChange: number,       // 来客变化%
  avgOrderChange: number,      // 客单变化%
  memberPenetration: number,   // 会员占比%
  ecommerceRatio: number,      // 电商占比%
  consecutiveDecline: number,  // 连续下降天数
  volatility: number           // 波动率
}
输出: {
  suggestedLabels: string[],
  rawData: object,
  status: 'pass'|'attention'|'warning'
}
```

候选标签池：`稳步增长 / 波动式增长 / 假增长 / 瓶颈 / 衰退 / 促销依赖 / 库存拖累 / 流量下滑但客单补偿`

算法预判规则：
- 营收↑ 毛利↑ → 候选"稳步增长"
- 营收↑ 毛利↓ → 候选"假增长"
- 来客↓ 客单↑ → 候选"流量下滑但客单补偿"
- 连续下降 ≥5天 → 候选"衰退"

最终标签由AI确认/调整。

---

#### `prepareGrowthDecomposition` ✅
**用途**：增长来源拆解（算法拆解，AI解读）

```
输入: rows: BizRow[] (当期 + 上期)
输出: {
  revenueChange: number,
  decomposition: {
    fromVisitor: number,
    fromAvgOrder: number,
    fromMember: number,
    fromEcommerce: number
  },
  aiPromptData: object,
  status: 'pass'|'attention'|'warning'
}
```

算法：
- `fromVisitor = (本期来客-上期来客) × 上期客单`
- `fromAvgOrder = (本期客单-上期客单) × 本期来客`
- `fromEcommerce = 本期电商 - 上期电商`

---

#### `prepareSalesQualityCheck` ✅
**用途**：销售质量判断（算法出数据，AI出结论）

```
输入: {
  revenueChangePct: number,
  grossProfitChangePct: number,
  grossMarginCurrent: number,
  grossMarginPrevious: number
}
输出: {
  revenueGrowsFasterThanProfit: boolean,
  marginDelta: number,
  qualityScore: number,
  aiPromptData: object,
  status: 'pass'|'attention'|'warning'
}
```

算法：
- `qualityScore` = 100 - |营收增速 - 毛利增速| × 权重
- `revenueGrowsFasterThanProfit` = 营收增速 > 毛利增速

#### `prepareChannelRiskAssessment` ✅
**用途**：渠道依赖风险评估

```
输入: {

  o2oTrend: object,                  // calcO2OTrend 输出
