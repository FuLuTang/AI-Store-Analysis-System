# Smolagents CodeAgent — 方法2

> 脏数据处理管线。CodeAgent 写 Python 动态展平 + DuckDB SQL 计算，应对深层嵌套和不规则输入。
> 与确定性管线（方法1）和 Pydantic AI 管线（方法3）并列。

---

## 定位

方法2，三条管线实现同一 `AgentPipeline` 接口，按数据质量/场景切换。

## 生产实现

已实现 `packages/agents/` 包，核心链路：

```
DatasetBundle → Workspace → write_raw_parquet → init_duckdb
  → build_smol_tools → CodeAgent → AgentResult
```

| 组件 | 说明 |
|------|------|
| `Workspace` | 持久化工作区 `storage/artifacts/{id}/`，带 duckdb/manifest/trace |
| `build_smol_tools(ws)` | 闭包注入 workspace 的 @tool 适配器 |
| `tools/impl/` | 三层架构：纯函数实现层 |
| `SmolPipeline` | 编排器：创建 workspace → 写 parquet → 初始化 DuckDB → 创建 CodeAgent → 收集结果 |
| `prompts/smol.md` | 系统提示词，引导 CodeAgent 用原生 Python 写文件操作 |
| `agent_route.py` | `POST /api/agent/analyze?pipeline=smol` |

详见 [架构设计.md](./架构设计.md)。

## 依赖安装

```bash
pip install "smolagents[litellm]" duckdb pandas
```

smolagents 本身是轻量框架，LLM 推理在远程 DeepSeek API 上，本地只跑 Python 代码生成和 DuckDB SQL 查询。2h2g 够用。

## Smolagent CodeAgent 工具体系

| 名字 | 类型 | 说明 | 共用 |
|------|------|------|------|
| SmolPipeline | 编排器 | 创建 workspace → 注入上下文/tools → 启动 CodeAgent → 收集结果 | False |
| prompts/smol.md | 流程指引 | 告诉 CodeAgent 用原生 Python 处理文件操作 | False |
| `build_smol_tools(ws)` | 适配器 | 闭包注入 workspace，生成 smolagents @tool 列表 | False |
| `setup_workspace` | tool | 扫描 parquet、注册 DuckDB、返回状态摘要 | True |
| `duckdb_query` | tool | 只读 SQL 查询 | True |
| `duckdb_register_parquet` | tool | 注册 parquet 为 DuckDB 表 | True |
| `read_context` | tool | 读上下文文档（指标定义/字段规则） | True |
| `profile_table` | tool | 读取 parquet 字段画像 | True |
| `validate_result` | tool | 校验输出是否符合 AgentResult schema | True |
| `list_tables` | tool | 查看 DuckDB 中已注册的表 | True |
| `read_plan` | tool | 读取任务清单（output/plan.json） | False |
| `check_plan` | tool | 标记步骤成功/失败 `check_plan(True, idx)` | False |
| `design_plan` | tool | 初始化写入 plan.json（编排器专用，不暴露给 Agent） | False |
| `cleanup_workspace` | tool | 清理大文件（完成后调用） | True |

CodeAgent 原生 Python 可直接做：`open().read()` / `open().write()` / `os.listdir()` / `pandas.read_parquet()` / `duckdb.connect()`。

## 评判标准

| 维度 | 期望 |
|------|------|
| 嵌套 JSON 展平 | 自己写递归展平，不依赖外部库 |
| 脏数据鲁棒性 | 空值 / 类型混用 / 错位字段能兜底 |
| SQL 生成 | 能用 DuckDB SQL 完成 ratio / group by 等基本指标 |
| 代码可靠性 | 同一输入多次跑，生成的代码和结果是否稳定 |
| 审计可追溯 | agent_trace.json / manifest.json / scripts/ 保留 |
