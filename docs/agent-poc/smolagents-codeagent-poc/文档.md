# Smolagents CodeAgent — 方法2

> 脏数据处理管线。利用 CodeAgent 写 Python 动态展平 + DuckDB SQL 计算，应对深层嵌套和不规则输入。

---

## 定位

方法2，与确定性管线（方法1）和 Pydantic AI 管线（方法3）并列。三条线实现同一 `AgentPipeline` 接口，按数据质量/场景切换。

```
脏数据 → CodeAgent 写 Python 展平 → DuckDB 查询 → 输出 metrics
```

## 优势

- 动态写代码，不需要预定义解析规则
- 脏数据鲁棒性强（空值、类型混用、错位字段都能兜底）
- DuckDB SQL 生成灵活，不需要注册计算器

## 生产实现

已实现 `packages/agents/` 包，核心文件：

| 文件 | 说明 |
|------|------|
| `smol_pipeline.py` | `SmolPipeline` 实现 `AgentPipeline` 接口 |
| `workspace.py` | `AgentWorkspace` 独立临时目录 |
| `tools/` | 6 个共享工具（file / duckdb / python / context / validate / profile） |
| `prompts/smol.md` | Smolagent 系统提示词 |
| `route: agent_route.py` | `POST /api/agent/analyze?pipeline=smol` |

详见 [架构设计.md](./架构设计.md)。

## Smolagent CodeAgent Tool Use

| 名字                       | 类型     | 说明                                                                                                  | 共用  |
| -------------------------- | -------- | ----------------------------------------------------------------------------------------------------- | ----- |
| CodeAgentRunner            | 编排器   | 启动 smolagents CodeAgent，执行基础操作如创建工作区、清理、创建环境...                                | False |
| Agent.md                   | 流程指引 | 告诉 CodeAgent 怎么工作、不能碰什么                                                                   | False |
| `read_workspace_file`      | tool     | 读本次上传文件                                                                                        | True  |
| `write_workspace_file`     | tool     | 写脚本 / manifest / 中间结果                                                                          | True  |
| `run_python_script`        | tool     | 在 workspace 沙箱内执行 Python 脚本，限制路径 / 超时 / 内存 / 网络                                    | True  |
| `duckdb_tool`              | tool     | 建临时库、注册 parquet、执行 SQL                                                                      | True  |
| `read_context_tool`        | tool     | 读指标文档、字段定义、行业规则                                                                        | True  |
| `validate_result_tool`     | tool     | 校验 manifest / mapping / metrics                                                                     | True  |
| `list_workspace_files`     | tool     | 查看当前目录文件                                                                                      | True  |
| `submit_final_result_tool` | tool     | 强制提交标准 AgentResult : { "tables_manifest": "...", "mapping": [], "metrics": [], "warnings": [] } | False |

## 评判标准

| 维度           | 期望                                             |
| -------------- | ------------------------------------------------ |
| 嵌套 JSON 展平 | 自己写递归展平，不依赖外部库                     |
| 脏数据鲁棒性   | 空值 / 类型混用 / 错位字段能兜底                 |
| SQL 生成       | 能用 DuckDB SQL 完成 ratio / group by 等基本指标 |
| 代码可靠性     | 同一输入多次跑，生成的代码和结果是否稳定         |
| 沙箱安全       | Docker 隔离，不挂载源码/.env/home                |
