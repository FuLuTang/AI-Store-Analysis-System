## 统一输出规范

所有指标返回：

```ts
type MetricStatus = "pass" | "attention" | "warning" | "uncountable";

type MetricResult = {
  metricId: string;
  name: string;
  value: any;
  unit?: string;
  status: MetricStatus;
  reason?: string;
  evidence?: any[];
  requiredFields: string[];
  missingFields?: string[];
  confidence: number;
};
```

规则：

- 缺必需字段：`uncountable`
- 除以零：`uncountable`
- 算值由系统完成
- 报警判断由 `Threshold Resolver` 完成
- AI 不直接写生产计算代码

## 指标注册表

```ts
type MetricDefinition = {
  metricId: string;
  name: string;
  requiredFields: string[];
  optionalFields?: string[];
  domains: string[];
  calculator: string;
  params?: object;
  healthProfiles: string[];
};
```

示例：

```yaml
metric_id: gross_margin_rate
name: 毛利率
required_fields: [revenue, gross_profit]
domains: [pharmacy, restaurant, retail]
calculator: ratio
params:
  numerator: gross_profit
  denominator: revenue
  unit: "%"
health_profiles: [default, o2o_driven, offline_driven]
```

## 通用计算器

### `ratio`

**用途**：比率类指标。

```text
value = numerator / denominator
```

例：

- 毛利率 = 毛利 / 营收
- 客单价 = 营收 / 订单数
- 会员占比 = 会员金额 / 总营收

---

### `period_change`

**用途**：环比 / 同比。

```text
changePct = (current - previous) / previous * 100
```

输入：

```ts
field: string;
periodField: date;
compare: "previous_period" | "same_period_last_year";
```

---

### `share_by_dimension`

**用途**：按渠道、商品、部门等维度计算占比。

```text
share = group_value / total_value
```

例：

- 美团占比
- 外卖占比
- TOP部门人数占比
- 菜品销售占比

说明：不写死美团/饿了么，统一按 `channel` 分组。

---

### `concentration`

**用途**：集中度。

```text
concentration = max(group_value) / total_value
```

例：

- 单平台依赖
- 单商品依赖
- 单部门人员集中

---

### `top_contribution`

**用途**：TOP N 贡献度。

```text
topNContribution = sum(topN value) / total
```

例：

- TOP3商品销售占比
- TOP5菜品销售占比
- TOP部门离职贡献

---

### `trend_slope`

**用途**：判断趋势方向。

```text
y = ax + b
slope = a
```

输出：

```ts
{ slope: number, direction: 'rising'|'falling'|'stable' }
```

---

### `anomaly_detect`

**用途**：异常点检测。

支持：

- 阈值法：环比超过指定比例
- z-score：`|z| > 2`
- IQR：箱线图异常

---

### `threshold_rate`

**用途**：超过阈值的占比。

例：

```text
配送超时率 = delivery_duration > 30分钟 的订单数 / 总订单数
```

## AI辅助项

AI只做辅助，不做核心计算。

### `prepareSceneContext`

用途：识别行业、业态、数据范围。

输出：`SceneContext`

---

### `prepareSemanticMapping`

用途：字段语义映射。

输出：`SemanticMapping[]`

---

### `prepareMetricSuggestions`

用途：发现额外可分析字段。

例：

```json
{
  "suggested_metric": "配送超时率",
  "required_fields": ["delivery_duration"],
  "calculator": "threshold_rate",
  "status": "pending_confirm"
}
```

说明：建议指标需用户确认或开发者注册后进入正式指标库。

---

### `prepareReportContext`

用途：把指标结果、证据、数据质量整理给报告模型。

输入：

```ts
{
  scene: SceneContext,
  metrics: MetricResult[],
  evidence: EvidenceItem[],
  dataQuality: object
}
```

## MVP指标优先级

### P0 通用必做

| 指标       | 需要字段                                   |
| ---------- | ------------------------------------------ |
| 营收趋势   | `date, revenue`                            |
| 环比/同比  | `date, revenue`                            |
| 客单价     | `revenue, order_count`                     |
| 毛利率     | `revenue, gross_profit`                    |
| 渠道占比   | `revenue, channel`                         |
| 渠道集中度 | `revenue, channel`                         |
| TOP贡献    | `revenue, product_name/department/channel` |
| 异常波动   | `date, revenue`                            |
| 数据完整度 | 任意数据                                   |

| 指标       | 需要字段                                     |
